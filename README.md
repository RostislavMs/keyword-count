# Підрахунок ключових слів

Веб-застосунок: бере текст із **публічного Google-документа**, рахує точні
входження заданих ключових слів/фраз і порівнює з потрібною кількістю.

## Стек

- Next.js 15 (App Router)
- React 19
- Tailwind CSS 4
- TypeScript
- Структура в `src/`

## Запуск

```bash
npm install
npm run dev      # http://localhost:3000
```

Для продакшну:

```bash
npm run build
npm start
```

## Як користуватися

1. Зробіть Google-документ публічним: **Поділитися → Будь-хто, хто має посилання → Глядач**.
2. Вставте посилання на документ.
3. Вставте дві колонки (ключ + кількість), наприклад скопійовані з таблиці:
   ```
   робота            5
   пошук талантів    3
   резюме            2
   ```
4. Натисніть **Порахувати**.

Результат для кожного ключа:

- `keyword - increase to N` — фактично менше за ціль, треба додати;
- `keyword - reduce to N` — фактично більше за ціль, треба зменшити;
- `keyword - ok (N)` — збігається з ціллю.

## Логіка підрахунку

- **Точний збіг**: ціле слово або точна фраза (без словоформ).
- Межі слова визначаються через Unicode-літери, тому коректно працює з
  кирилицею: «робота» не зараховується в «роботи» чи «роботодавець».
- За замовчуванням без урахування регістру (можна увімкнути перемикачем).

## Структура

```
src/
  app/
    api/analyze/route.ts   # серверне завантаження документа + підрахунок
    layout.tsx
    page.tsx               # UI
    globals.css
  lib/
    keywords.ts            # розбір ключів і логіка підрахунку
```

## Примітки

- Документ читається серверно (через `export?format=txt`), тому проблем з
  CORS немає.
- Посилання типу «Опубліковано в інтернеті» (`/document/d/e/...`) не
  підтримуються — використовуйте звичайне посилання спільного доступу.
